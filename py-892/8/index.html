Що таке Класи і як ними користуватись

Чим тротуар схожий на тигра? Тим, що тротуар і тигр - це сутності (інакше кажучи - певні поняття реального світу). В розмовній мові вони є іменникиками, а в Пайтоні - об'єктами.

Об'єкти - це спосіб організації коду в програмі, а ще спосіб розділення складної задачі на прості маленькі.

Щоб як слід розібратись в тому, що таке об'єкти, треба почати з типів. Почнемо з тигрів і тротуарів.

Тигр - це савець (з біології), а савці це складова тварин. Окрім того тигр - істота з душею, так як він живий.

Тепер про тротуар. Головне, що ми можемо сказати про тротуар - він нежививий, а отже відноситься до неживих об'єктів. Тобто "Савці", "Тварини", "Живі", "Неживі" - це спосіб класифікації об'єктів реального світу.

Розділяємо об'єкти на класи

У мові Python об'єкти визначаються класами, завдяки яким об'єкти можна розділити за сенсом. Ось діаграма класів, яка відповідає нашим уявленням про тротуари та тигрів.

Основний клас це Сутності. Він має два підкласи = Неживі та Живі. До класу Неживі відноситься підклас Тротуари. У класу Живі - підклас Тварини, у якого підклас Савці, а у того в свою чергу підклас Тигри.

А поки давайте створимо класи, які щойно обговорили.

class Things:
pass

Ми дали ім'я класу Things і використали конструкцію pass, яка визначає, що більше ніякої інформації всередині класу ми вказувати не будемо. Pass - команда, за допомогою якої можна створювати класи і функції, не програмуючи їх поведінку.

Предки і Нащадки

Коли один клас є підкласом іншого, кажуть, що перший клас - Нащадок іншого (як "дочірній" клас), а другий - його Предок (як "батьківський" клас). Один і той самий клас може бути як нащадком декількох класів, так і предком для декількох інших. На нашій діаграмі клас, що знаходиться над іншим класом - його предок, а клас під ним - його нащадок. Наприклад класи Inanimate і Animate - нащадки класу Things, який є їх предком.

Для того щоб визначити, що клас, який ми створюємо є нащадком іншого класу, треба вказати ім'я класу-предка в дужках після імені нового класу. Ось так:

class Inanimate(Things):
pass

class Animate(Things):
pass

Тут ми створили клас Inanimate, вказавши, що він є нащадком класу Things, далі створили так само клас Animate.

Тепер давайте створимо клас Sidewalks, вказавши його предком класу Inanimate:

class Sidewalks(Inanimate):
pass

І нарешті, аналогічним способом створимо класи Animals, Mammals i Tigers, не забувши вказати їх предків:

class Animals(Animate):
pass

class Mammals(Animals):
pass

class Tigers(Mammals):
pass

Створюємо об'єкти для класів

Тепер, коли у нас є набір клаів, саме час створити об'єкти для цих класів. Нехай у нас є тигр, якого звати Алекс. Ми знаємо що він відноситься до класу Tigers. Для того щоб "навчити" цього Пайтон, потрібно написати наступний код:

alex = Tigers()

Це означає: створити об'єкт класу Tigers і присвоїти його змінній alex. Після імені класу ставляться дужки, як і після імені функції.

Визначення атрибутів та методів класу

Всередині класу можна визначити змінні, або атрибути класу. Вони описують певні характеристики (вік, ріст, вагу, розміри, ім'я, тощо) об'єкта цього класу і можуть використовуватися всередині методів (функцій) класу та його нащадків. Методи (тобто функції, що створені всередині класу) задають дії, які може виконувати об'єкт цього класу та його нащадки.

class Animals(Animate):
age = 10
weight = 20

def eat(self):
print('їсть')
weight+=2

Використовуємо функції для задання характеристик класу

Роздивимось наш клас Animate та його нащадків. Можна кожному з них задати характеристики. Характеристики - особливості, які властиві усім об'єктам цього класу, а також їхнім об'єктам-нащадкам.

Наприклад, що спільного у всіх тварин? Вони дихають, рухаються і харчуються. А у савців? Вони годують своїх дітей молоком, а також дихають, рухаються і харчуються. Тигри, як нам відомо, хижі тварини, отже полюють на інших тварин а також, як і всі савці годують дітей молоком, як тварини дихають, рухаються і харчуються.

Ці характиристики можна уявити як дії, або функції - те, що об'єкт або клас може робити.

Для додавання функцій класу ми використовуємо def, тому клас Animals буде виглядати так:

class Animals(Animate):
def breathe(self):
pass

def move(self):
pass

def eat(self):
pass

В першому рядку ми визначили клас - так само, як робили це раніше. Але наступним рядком замість pass дали визначення функції breathe з одним параметром self. Параметр self дає функції класу можливість викликати інші функції цього класу, а також його предків.

Давайте тепер створимо функції для нащадків:

class Mammals(Animals):
def feed_baby_milk(self):
pass

class Tigers(Mammals):
def hunting(self):
pass

Навіщо потрібні об'єкти і класи?

Ми надали нашим класам функції, але навіщо потрібні класи і об'єкти, якщо можна було просто записати звичайні функції?

Пояснити це нам допоможе тигр Алекс, якого ми створили, як об'єкт класу Tigers:

alex = Tigers()

Оскільки Алекс є об'єктом класу Tigers, ми можемо використовувати функції, які визначені в його класі Tigers та в класах предках. Робиться це наступним чином:

alex = Tigers()
alex.move()
alex.hunting()

Тепер уявимо, що у Алекса є друг тигр - Федір. Давайте створимо ще один об'єкт класу Tigers

fedir = Tigers()

Оскільки ми використовуємо об'єкти і класи, при виклику функції ми можемо вказати до якого тигра ми використовуємо функцію. Наприклад, щоб вирішити, хто повинен рухатися - Алекс чи Федір.

Давайте трохи змінимо наші функції, щоб побачити це:

class Animals(Animate):
def breathe(self):
print('Дихає')

def move(self):
print('Рухається')

def eat(self):
print('Їсть')

class Mammals(Animals):
def feed_baby_milk(self):
print('Кормить молоком')

class Tigers(Mammals):
def hunting(self):
print('Полює')

Тепер коли ми створимо об'єкти Алекс і Федір і будемо викликати для них функції, буде видно, що відбувається:

alex = Tigers()
fedir = Tigers()
alex.move() # Рухається
fedir.hunting() # Полює

Якби це були справжні тигри, а не об'єкти, то один з них рухався б, а другий полював.

Функції, які викликають інші функції

Викликаючи функцію об'єкта, ми використовуємо ім'я змінної, в якій цей об'єкт знаходиться. Наприклад, так ми викликали для тигра Алекса функцію "рухатися":

alex.move()

Якщо ж потрібно викликати функцію "рухатися" з іншої функції, визначеної в класі Tigers, замість імені змінної потрібно використовувати аргумент self. Таким чином можна викликати будь-яку функцію класу (або класів-предків) із іншої функції класу. Наприклад, додамо в клас Tigers функцію find_eat:

class Tigers(Mammals):
def find_eat(self):
self.move()
self.hunting()
print('Я дістав їжу')
self.eat()

Ініціалізація об'єктів

При створенні об'єкту іноді необхідно задати якість значення (властивості), які знадобляться для роботи з цим об'єктом. Підготовка об'єкту для використання називається інціалізацією.

Наприклад, ми хочемо вказати кількість смужок на шкурі тигра при створенні кожного об'єкту класу Tigers. Для цього потрібно створити функцію з ім'ям __init__. Функція з таким іменем має особливе значення для класів - вона дозволяє задати властивості об'єкта при його створенні, оскільки для кожного нового об'єкту вона викликається автоматично. Ось як користуватись цією функцією:

class Tigers(Mammals):
def __init__(self, lines):
self.tigers_lines = lines

Написавши def __init__(self, lines):, ми створили функцію ініціалізації, яка приймає два аргументи, self i lines. Як і в інших функціях класів, функція ініціалізації має приймати значення self першим аргументом. В блоці функції рядок self.tigers_lines = lines присвоює значення аргумента lines властивості об'єкту tigers_lines. Таким чином, цей рядок значить "Взяти значення параметру lines і помістити його в змінну об'єкта з іменем tigers_lines".
